CONTEXTO

Este é o site institucional da Sensorama Design, uma consultoria
brasileira de design e inovação. O projeto foi migrado de Figma Make
(SPA React puro com problemas de SEO) para Next.js 14 com App Router.

A ESTRUTURA BASE já está pronta:
- Configurações (package.json, next.config, tailwind, tsconfig)
- Layout raiz com SEO completo, Schema JSON-LD e Analytics
- Rotas multilíngue PT/EN/ES via App Router
- Sitemap.xml e robots.txt dinâmicos
- Dados completos dos 3 cases visíveis (PT/EN/ES) em src/lib/cases-data.ts
- CaseDetailPage e CasesListingPage funcionais

O QUE VOCÊ PRECISA FAZER

1. Completar a LandingPage (componente em src/components/LandingPage.tsx)
2. Completar as páginas legais (PrivacyPolicyPage, CookiePolicyPage, TermsOfUsePage)
3. Criar componentes faltantes (ContactModal, CookieBanner, ConsentManager)
4. Implementar SVG heroes customizados por case
5. Ajustar e organizar imagens em public/images/
6. Testar build e indexação

TAREFA 10: DEPLOY

10.1. GitHub:
      Criar repositório novo "sensorama-site" na conta GitHub da Sensorama.
      git init, add, commit, push

10.2. Vercel:
      Criar conta Vercel com email da Sensorama (contato@sensoramadesign.com).
      Importar projeto do GitHub.
      Configurar variáveis de ambiente do .env.example
      Deploy automático.

10.3. Domínio (GoDaddy):
      
      Acessar painel GoDaddy do domínio sensoramadesign.com.
      
      Mudanças no DNS:
      
      REMOVER:
      - CNAME www → sites.figma.net  (era do Figma Make)
      
      ADICIONAR (Vercel dará os valores exatos):
      - CNAME www → cname.vercel-dns.com
        ou
      - A @ → 76.76.21.21 (IP da Vercel)
      
      MANTER:
      - Redirect sensoramadesign.com → www.sensoramadesign.com (já configurado)
      - CNAME sensoai → ghs.googlehosted.com (ferramenta do outro time, NÃO MEXER)
      - MX records do Google (email)
      - TXT records (verificações)
      
      Aguardar propagação DNS (1-4h, às vezes até 48h)

10.4. Google Search Console:
      
      - Validar propriedade do domínio (via DNS é mais robusto)
      - Submeter sitemap: https://www.sensoramadesign.com/sitemap.xml
      - Solicitar reindexação das páginas principais
      - Aguardar reindexação (alguns dias)

   - Todo conteúdo de página principal precisa estar no HTML estático

   - Performance e Core Web Vitals importam

   - Casos são anonimizados ("organização de healthcare", etc.)
   - Não introduzir nomes de empresas

DÚVIDAS COMUNS
============================================================

Q: Posso mudar o design?
A: Não nesta migração. O objetivo é resolver SEO mantendo o design atual.
   Melhorias de design ficam para uma próxima fase.

Q: E se faltar uma imagem?
A: Use um placeholder visualmente coerente (fundo cinza com texto, ou
   cor da brand) e marque com comentário // TODO: adicionar imagem.

Q: Posso adicionar páginas novas?
A: Sim, depois que tudo estiver funcionando. Por agora, focar em
   espelhar o que existe no Figma Make.

Q: O que fazer com o site antigo no Figma Make?
A: Depois que o novo estiver no ar e indexado, despublicar o antigo.
   Manter o projeto Figma Make como histórico.

Q: Precisa migrar o GoDaddy também?
A: Não. O domínio fica na GoDaddy. Só muda o DNS para apontar para Vercel.
